# 本机 Surge 测试与模块加载说明

这份说明用于本机开发测试。正式测试优先从自托管 GitLab raw URL 安装模块。模块按网站分类，用户可以按需加载单个网站模块。

## 重要结论

- Surge CLI 当前没有 `module add` / `module enable` 命令，不能直接安装模块。
- Surge CLI 可以验证脚本运行面：`script evaluate`。
- Surge GUI 添加模块需要 URL，本机测试使用本地 HTTP 服务提供模块 URL。
- 不要直接用 `/modules/<site>.sgmodule` 的本地文件路径安装。使用 `/local/modules/<site>.sgmodule` 生成的本机专用 URL。
- 本机专用模块会自动把相对路径或远程 raw `script-path` 改成 `http://127.0.0.1:8787/scripts/dist/*.js`。
- 测试本地模块期间保持本地 HTTP 服务运行。

## 本机 URL 服务

先启动本地模块服务。推荐使用 LaunchAgent 常驻方式：

```bash
cd /Users/yara/Developer/Proxy/modules
./tools/start-local-module-server.sh
```

停止本地模块服务：

```bash
cd /Users/yara/Developer/Proxy/modules
./tools/stop-local-module-server.sh
```

也可以在 Terminal 前台临时运行：

```bash
cd /Users/yara/Developer/Proxy/modules
node tools/local-module-server.mjs
```

默认监听：

```text
http://127.0.0.1:8787
```

Surge GUI 的 Modules 添加 URL 时，使用下面这些本机 URL：

```text
http://127.0.0.1:8787/local/modules/linuxdo.sgmodule
http://127.0.0.1:8787/local/modules/psnine.sgmodule
http://127.0.0.1:8787/local/modules/keylol.sgmodule
http://127.0.0.1:8787/local/modules/gamer.sgmodule
```

注意：

- 这些 URL 只用于本机测试。
- 测试期间必须保持本地模块服务运行，否则 Surge 会出现“无法连接服务器”或“无法获取脚本”。
- 如果 8787 端口被占用，可以换端口：

```bash
node tools/local-module-server.mjs --port 8790
```

然后在 Surge GUI 中使用 `http://127.0.0.1:8790/local/modules/<site>.sgmodule`。

如果 Surge 提示“无法获取脚本”，先在 Terminal 验证脚本 URL：

```bash
curl -fsS http://127.0.0.1:8787/scripts/dist/psnine.js | head -n 1
```

正常会看到：

```text
/* Auto-generated by tools/build-modules.mjs. Edit scripts/core or scripts/adapters instead. */
```

## Surge App 加载流程

1. 打开 Surge Mac。
2. 进入当前 Profile 的 Modules。
3. 添加需要的网站模块 URL，例如：
   `http://127.0.0.1:8787/local/modules/linuxdo.sgmodule`
4. 启用模块。
5. 确认 `Scripting` 已启用。
6. 确认 `MITM` 已启用，并包含模块声明的 hostname。
7. 登录目标网站，访问抓包入口。
8. 手动运行对应脚本一次，看通知和日志。

## 上传 GitLab 前检查

每次修改 core、adapter、module 或文档后运行：

```bash
cd /Users/yara/Developer/Proxy/modules
node tools/build-modules.mjs
node tools/check.mjs
python3 /Users/yara/.agents/skills/check/scripts/audit_signals.py --root /Users/yara/Developer/Proxy/modules
```

`node tools/check.mjs` 会自动检测本机 Surge CLI，并运行 `script evaluate` 验证 dist 脚本能在 Surge runtime 启动。

## GitLab 发布建议

- 保留 `modules/<site>.sgmodule` 按网站分类。
- 保留 `scripts/dist/<site>.js` 作为用户可直接加载的脚本。
- README 中给每个网站单独列模块 URL，用户按需导入。
- 不要把本机绝对路径写入公开版 `.sgmodule`。
- 公开版 `script-path` 应使用 GitLab raw URL，例如：
  `http://192.168.3.11:5580/yara/surge-auto-sign-modules/-/raw/main/scripts/dist/linuxdo.js`

## Anyrouter.top 约束

不要测试、自动化或发布 anyrouter.top 自动打开模块。该站点会触发 WAF/限流，详见项目根目录 `AGENTS.md`。
